DePaul Gaming Lab Inventory System

In my time with DePaul University working in the gaming labs, I built a full Java Swing inventory and sales system connected to a MySQL database. The swing interfaces is built upon a JDBC controller system. The controller has a built in log system when placed into logging mode. It also has hooks into the JavaMail API for sending low or wrong stock messages to the moderating staff.

As the only developer, it was my responsibility to implement all user requests. This, over time, included an expandable game list, a simple sales system, a medals (think achievements) system, a user warning system, and an administrative panel.
